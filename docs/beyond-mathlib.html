Beyond Mathlib: proving the axiomatized parts for real

Where the formalization stands

The Lean development closes every statement of the paper with zero sorry – the headline theorems (1.1, 1.2), the propositions (2.1, 2.2, 3.1, 4.1, 4.2), and the mid-level lemmas all compile. But closing a statement and proving it from first principles are different claims, and this project is scrupulous about the difference.

The self-contained cores – the derivative computations, the geometry, the Grönwall / Lyapunov / pigeonhole leaves (L1–L11) – carry a clean kernel proof. The deep content that Mathlib lacks was stated as a layer of clearly labeled axioms; “proving the axiomatized parts for real” means building that missing mathematics in Lean and discharging them, so a node’s effective status climbs axiomatised → machine-checked (a node inherits the minimum status over its dependency closure).

That program is now well underway. The original infrastructure axiom layer is discharged: the optimal-transport / Wasserstein theory (M2) and the per-block characteristic flow (M3) are built from Mathlib’s Picard-Lindelöf, Grönwall, and coupling machinery, and are machine-checked. A statement-fidelity audit (findings F11–F16 in RESEARCH.md) then sharpened the picture in an important way: the M3 flow is a linear (measure-independent) pushforward, and the paper’s own eq. (1.7) proves such dynamics cannot disentangle overlapping measures – so the measure-dependent statements were restated over a mean-field attention layer (M3b): the paper’s velocity field (1.2) as a concrete definition, with a well-posedness axiom pair (exists_meanFieldFlow / meanFieldFlow_unique) as the honest McKean-Vlasov gap – of which the uniqueness half (meanFieldFlow_unique) is now a machine-checked theorem (a measure-averaged Grönwall argument on the sphere), leaving only existence (exists_meanFieldFlow) axiomatised. Today #print axioms theorem_1_1 lists exactly that existence axiom plus the paper-level dynamical statements (cluster_to_point, exists_disentangling_balls, exists_parked_schedule; theorem_1_2 swaps cluster_to_point for lemma_5_4), each encoding a result from the paper’s Sections 2–5 that rests on classical theorems (LaSalle invariance, Hartman-Grobman linearization) Mathlib does not yet have. This page is the public roadmap for that program – and, increasingly, a record of it.

Progress so far

  • M2 – Optimal transport / Wasserstein: done (machine-checked). Axioms.W2 is no longer opaque: it is now the definition (Foundations.W2 μ ν).toReal over the ENNReal Kantorovich coupling cost, and its five structural facts – nonnegativity, symmetry, the triangle inequality, the map / L² displacement bound, and convexity along mixtures – are proved theorems. Because toReal sends ⊤ to 0, the honest ℝ-valued statements carry exactly the finiteness / measurability hypotheses that keep the ENNReal→ℝ bridge sound, discharged at the call sites from bounded (sphere) support.
  • M3 – Per-block characteristic flow: done (machine-checked). The flow flowMap is built as the fold of Picard-Lindelöf solutions of a Block – a globally-Lipschitz, globally-bounded, radially-tangent velocity field with a duration. Its Lipschitz, bijectivity, semigroup (comp / id / reverse-and-negate inv), sphere-invariance, and parked laws are all derived from Mathlib’s IsPicardLindelof.of_time_independent + Grönwall, with the sphere kept invariant by the radial-tangency identity. This layer is linear in the measure – the honest home for the paper’s V ≡ 0 (perceptron-only) sections (§4, Appendix B).
  • M3b – Mean-field attention layer: uniqueness machine-checked, existence axiom (findings F14, F20). The fidelity audit kernel-refuted the family-level statements when read over the linear flow – two identical inputs cannot enter disjoint balls under one pushforward map, exactly the paper’s eq. (1.7) obstruction. The fix is structural: the paper’s measure-dependent velocity (1.2) – the softmax attention average plus perceptron, tangentially projected – is now a concrete definition (AttnParams, attnAvg, AttnParams.field), a mean-field flow is a predicate (IsMeanFieldFlow), and the well-posedness of that flow was an axiom pair – existence and uniqueness. The uniqueness half (meanFieldFlow_unique) is now a discharged theorem: found unsound off the sphere (F20) and repaired to the paper’s μ₀ ∈ P(𝕊^{d-1}) datum, then closed by a decomposed campaign – the field’s full joint (point, W₁)-modulus, an integral-form Grönwall, an FTC representation, a Fubini-averaged distance, and an ODE_solution_unique transfer. Only existence (exists_meanFieldFlow) – McKean-Vlasov well-posedness on the sphere, which additionally needs Picard-Lindelöf plus Wasserstein completeness of \(\mathcal P(\mathbb S^{d-1})\) – remains axiomatised, the true remaining M3 gap. All measure-dependent statements (Prop 2.1, Lemma 3.3, the disentanglement, Lemmas 5.4, parking, Theorems 1.1/1.2) now live on this layer. The two layers are bridged: a V = 0 attention step is the linear block-flow pushforward (attnStep_eq_map_blockFlow, the first consumer of meanFieldFlow_unique), so the machine-checked Appendix-B transport acts on the mean-field layer as well.
  • M4 – Per-point transport (Appendix B): done (machine-checked). The full mass-transport chain is discharged: lemma_B_2 (the single overlapping ball pair) is a proved theorem, and with it lemma_B_1’s ball-chain induction reports a footprint of exactly propext / Classical.choice / Quot.sound. The proof recenters the self-centered gated machinery at a point of the cap overlap (the geodesic triangle inequality comes free from Mathlib’s angle_le_angle_add_angle) and an amplitude-scaled gated block realizes the paper’s ‖θ‖ ~ C/(Tε) parameter freedom at any fixed horizon. The admission review restored two more load-bearing hypotheses (a probability measure; sub-hemisphere radii R < π/2 – a larger cap can contain both ω and -ω, where every gated field vanishes) before the discharge. Built on: the ReLU-gated cutoff velocity field of B.5 as a genuine well-posed Block (the quadratic tangential projector made globally Lipschitz by a bespoke vanishing-gluing argument), the finite-time logistic reaching estimate (B.7), and the pushforward mass-retention bridge (B.8).
  • M8a – Atomless splitting: done (machine-checked). exists_atomless_partition (the disjoint, prescribed-mass decomposition behind Proposition 2.2) is fully de-axiomatized. The measure-theoretic primitive it rested on – Sierpiński’s intermediate-value theorem for nonatomic measures – is now a kernel-checked theorem: proved on ℝ (the cumulative t ↦ μ(E ∩ (-∞,t]) is continuous because its increment is the Bochner primitive ∫ 𝟙_E, valid because μ has no atoms, then the intermediate value theorem), and lifted to any standard Borel space by pushing the measure forward along a measurable embedding into ℝ. #print axioms on exists_atomless_partition and the IVT lists only propext / Classical.choice / Quot.sound – no custom axioms, no sorry. A soundness bug was surfaced and fixed along the way (see below).
  • M5 – Geodesic convexity on the sphere: done (machine-checked). The full stack: a GeodesicConvex predicate (closure under normalized great-circle chords), open hemispheres and strict spherical caps are geodesically convex, the hull characterization (geodesicHull = cone ∩ sphere is geodesic-convex and minimal), and the Section 3.3 separation transfer the paper uses without proof: clusters shrunk into r-balls around 2r-separated unit directions have disjoint geodesic hulls and non-colinear barycenters (geodesicHull_disjoint_of_separated_balls, barycenter_not_sameRay_of_separated_balls, the strict barycenter-level bound via an a.e. argument). The paper axiom exists_disentangling_balls (mean-field layer, M3b) now owes only dynamics facts and the family composition of Lemmas 3.2-3.4, no geometry.

Everything above is kernel-clean; the claim graph badges each node live, and the commit history records every step under the Conventional Knowledge Commits discipline (each Status: footer cross-checked against #print axioms by a pre-commit hook).

The leaves are written Mathlib-style from the start (snake_case terms, UpperCamelCase types, everything namespaced under MeasureToMeasure.*), so the clean ones are born re-usable. The status the badges read is regenerated from #print axioms and compared, which is the real drift guard: a blueprint checkdecls only confirms each \lean{} name exists, not that its stated status still matches the kernel. The full re-usability discipline is catalogued in the lean-math plugin’s reusable-blueprints.md.

What’s left, and how to finish

11 named axioms remain. They fall into three groups by how each is closed:

Tier 1 – constructive assembly (decomposition-tractable; the pieces already exist).

  • lemma_3_4_part1 (make two measures non-colinear) – the active front, and the one mid-level axiom on the linear layer (\(V \equiv 0\), so the field never reads the measure). Its App. B.3 Part 1 argument is mass-collapse + pigeonhole (not the barycenter linearization of Part 2), and its skeleton is now machine-checked: barycenter-of-a-flowed-measure (barycenter_measureFlow), the gated block’s off-cap parking / fixing clause (flowMap_gatedBlock_id_of_inner_le), and the separation pigeonhole itself (exists_mem_ball_barycenter_collapse_ne, on leaf L10). What remains is the mass-gap ball, a single-gated-block localized collapse to the pigeonhole point (reusing gatedBlock_reach, sidestepping the F16 non-localized ball-chain gap), and barycenter \(W_2\)-continuity – then it discharges, taking the count to 10.
  • exists_parked_schedule (parking) – the gated field vanishes off its cap, so a schedule parks disjoint supports; what remains is the non-interference argument plus the concatenation budget.

Blocked – constructive in principle, but stalled on a missing Mathlib layer (the continuity-equation / mass-sweep theory).

  • prop_2_2 (discrete clustering) – the mixture-combination core is machine-checked (measureFlow_W2_discrete_of_perPiece) and a cap-localized refinement is buildable, but the routing is the wall: prop_2_2 asks for a single Params d, and one deterministic flow map cannot split a cap’s mass toward several far-apart targets. The paper’s proof is a sequential \(O(M)\)-switch mass-sweep, which needs the continuity-equation theory Mathlib lacks (mapped in a scouting pass; the naive “one cap per piece” partition is false).
  • prop_4_2 (steering) – the geodesic-gradient corridor construction, the hardest constructive target. Closing it also makes prop_4_1 (already a theorem) kernel-clean and unblocks cluster_to_point.

Tier 2 – infrastructure builds (a genuine new layer of Mathlib-level theory).

  • exists_meanFieldFlow (the mean-field existence half) – Picard-Lindelöf plus Wasserstein completeness of \(\mathcal P(\mathbb S^{d-1})\). The Lipschitz-in-\((x, W_1)\) moduli are already machine-checked from the uniqueness campaign, so the same leaf-decomposition applies.
  • prop_2_1, cluster_to_point – the M6 LaSalle/Lyapunov convergence theory.
  • lemma_5_4 – the M7 \(L^2\) approximation of a transport map by flow maps.
  • lemma_3_3, lemma_3_4_part2 – the §3.3 mean-field barycenter shrink dynamics (Part 1 is on the linear layer and sits in Tier 1 above).

Top-level assembly.

  • exists_disentangling_balls – assembles the §3.3 chain and closes Theorems 1.1 / 1.2; it discharges last, once its dependencies land.

The method. The fork-sized leaf decomposition that discharged meanFieldFlow_unique – where six monolithic attempts stalled on context budget alone – is the template for the rest: cut the proof at its lemma boundaries, bank each leaf to main kernel-clean, and let the next cite the last instead of re-deriving it.

What must be built, and what Mathlib already gives

A three-part audit – the pinned Mathlib, the wider Lean 4 ecosystem, and cross-assistant (the Isabelle/HOL AFP and Coq/Rocq) – mapped the boundary precisely. The gaps are exclusively the optimal-transport and mean-field layers; the measure-theoretic substrate beneath them is mature.

What we can reuse

Nothing is importable across proof assistants, but the external audit found real leverage:

  • LaSalle / Lyapunov (M6) – a complete Coq proof (Cohen–Rouhling, ITP 2017, drouhling/LaSalle) is a proof-structure blueprint, and mcdoll/DynamicalSystems (Apache-2.0) ships Lean LaSalle.lean / Lyapunov.lean plus Carathéodory ODE extensions to vendor.
  • Optimal transport (M2) – absent in every assistant, so built genuinely from scratch (done): a coupling-based \(W_2\) over the ENNReal Kantorovich cost, from which the bound the paper needs is a one-liner (the pair map is an admissible coupling; Santambrogio, Optimal Transport for Applied Mathematicians, Ch. 5).
  • Sierpiński splitting (M8a) – absent everywhere; built (done) via the cumulative-distribution + intermediate-value-theorem route rather than Fremlin’s exhaustion argument, which turned out cleaner in Mathlib.
  • Geodesic convexity (M5) – Mathlib’s Riemannian-geometry substrate is landing in open PRs, but geodesic convexity on the sphere is absent; built in-repo the elementary way (normalized great-circle chords, pure inner products), the foundation now machine-checked (in progress).

A caveat kept in mind: several “0 sorry” AI-generated Lean repos for LaSalle/Lyapunov exist, but such artifacts frequently prove weakened statements – so any reuse is gated on a #print axioms check and a statement-faithfulness review, the same discipline this project applies to its own claims.

What we could give back: ForMathlib candidates

The reuse audit runs both ways. Several of the machine-checked leaves are general-purpose spherical geometry with no dependence on the paper’s specific construction, so they are natural ForMathlib candidates (marked in the Lean source with a -- ForMathlib candidate: comment):

  • the tangential-projector identities P_x^⊥ v = v - ⟪x,v⟫ x (symmetry, idempotence, and the quadratic identity ⟪P_x^⊥ v, v⟫ = ‖v‖² - ⟪x,v⟫²);
  • geodesic distance d_g(x,y) = arccos⟪x,y⟫ with its range and cos-of-angle facts;
  • the separating-hyperplane bound from cosine monotonicity;
  • barycenter non-colinearity for disjoint geodesic hulls.

These are prepared, not published: staging them ForMathlib-style (minimal imports, generic names, an Apache header, passing linters) is a mechanical readiness step, but actually contributing them to Mathlib is a deliberate human decision, not something the pipeline does on its own. The optimal-transport and mean-field-flow layers (now built in-repo, M2 / M3) are broader developments whose upstreaming would be a much larger, separate effort than these self-contained geometry leaves.

The first target: M8a – discharged

The chosen first slice was the smallest self-contained win: discharge exists_atomless_partition (the disjoint, prescribed-mass decomposition behind Proposition 2.2) by building the nonatomic splitting theorem. It needs no optimal-transport or flow prerequisites, which is why it started the program – and it is now complete and machine-checked (see Progress so far above). The pieces are normalized restrictions (αₖ)⁻¹ • μ|_{Aₖ} to a disjoint partition carved by iterating the Sierpiński IVT; that IVT, absent from Mathlib, was built from scratch on NoAtoms + the intermediate value theorem.

A soundness bug, surfaced while planning – and fixed

The axiom as originally stated was inconsistent. It demanded a decomposition whose every piece sits in an open hemisphere – but at \(M = 1\) that forces the whole measure into a half-space, which no centrally-symmetric atomless measure (a Gaussian; the uniform law on a ball or sphere) satisfies. The fix drops the hemisphere clause from the partition and acquires it dynamically per piece, the way the paper actually does. A second soundness point: the splitting is only true on a standard Borel space, not for bare NoAtoms (a countable–cocountable 0/1 measure is a counterexample) – so the proved theorem carries that hypothesis, which Eucl d satisfies. These are the latest of several fidelity corrections the formalization has forced – the entire point of running the argument through a kernel.

Scale and honesty

Two honesty subtleties the per-node badges cannot see. First, a hand-written axiom layer must be jointly consistent. A per-node #print axioms check is blind to a contradiction spread across several axioms – an inconsistent block would let the kernel prove anything while every badge still reads clean. The M8a soundness bug (the M = 1 hemisphere collapse, above) was exactly such a latent inconsistency, caught by reasoning about a model rather than by any single #print axioms. Second – the lesson of the 2026-07-03 fidelity audit (findings F11–F16) – each axiom must be transcription-honest: the kernel checks the statement as written, never that it matches the cited source. Eleven of the twelve axioms were false as first stated because the type-correct stubs had silently dropped the paper’s hypotheses; several fell to three-line kernel refutations. The repo now enforces the third honesty mechanically: every axiom is admitted through the axiom admission protocol (verbatim source anchor, hypothesis-parity diff, adversarial degenerate instantiation, model-adequacy check – see WORKFLOW.md), carries Source-Ref: / Refutation-Attempt: commit footers (CKC spec v0.2.0, rule 14, enforced by the pre-commit hook), has a per-claim fidelity record gated by the claim-graph audit, and leaves a refutation regression suite behind: the disproofs of the refuted drafts are committed as kernel-checked theorems, paired with must-fail adapter files that a CI gate compiles on every push – if an adapter ever compiles, an axiom has been re-loosened to a shape already proved false.
